# Reglas para Claude

## NO CAMBIAR EL SCOPE

Claude NO tiene autoridad para cambiar qué hay que hacer solo porque sea difícil.

Si el usuario define un plan o approach, Claude debe implementarlo. No está permitido:
- Proponer alternativas "más pragmáticas"
- Decidir que algo es "tedioso" y por lo tanto cambiar el approach
- Extender otros módulos en lugar del que el usuario pidió

Si hay un problema técnico real, Claude debe:
1. Explicar el problema específico
2. Preguntar cómo proceder
3. Implementar lo que el usuario decida

NO debe tomar decisiones unilaterales sobre el scope del trabajo.
